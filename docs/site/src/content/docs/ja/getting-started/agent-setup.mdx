## 仕組み

Mutsumi は [Agent Skills オープン標準](https://agentskills.io) に準拠した **5 つのバンドルスキル**を同梱しています。`mutsumi setup` を実行すると、これらのスキルファイルがシンボリックリンクで Agent のスキルディレクトリにインストールされます。Agent は自動的にスキルを検出するため、手動でプロンプトをペーストする必要はありません。

```
~/.mutsumi/skills/          ← SSOT（信頼できる唯一の情報源）
  mutsumi-manage/SKILL.md
  mutsumi-track/SKILL.md
  mutsumi-plan/SKILL.md
  mutsumi-report/SKILL.md
  mutsumi-context/SKILL.md
        │
        ├── symlink → ~/.claude/skills/     (Claude Code)
        ├── symlink → ~/.agents/skills/     (Codex CLI)
        ├── symlink → ~/.gemini/skills/     (Gemini CLI)
        └── symlink → ~/.config/opencode/skills/  (OpenCode)
```

## クイックスタート

<Steps>

1. Mutsumi をインストール（まだの場合）：

   ```bash
   pip install mutsumi-tui
   ```

2. Agent に対して setup を実行：

   <Tabs>
     <TabItem label="Claude Code">
     ```bash
     mutsumi setup --agent claude-code
     ```
     スキルは `~/.claude/skills/` にインストールされます。
     </TabItem>
     <TabItem label="Codex CLI">
     ```bash
     mutsumi setup --agent codex-cli
     ```
     スキルは `~/.agents/skills/` にインストールされます。
     </TabItem>
     <TabItem label="Gemini CLI">
     ```bash
     mutsumi setup --agent gemini-cli
     ```
     スキルは `~/.gemini/skills/` にインストールされます。
     </TabItem>
     <TabItem label="OpenCode">
     ```bash
     mutsumi setup --agent opencode
     ```
     スキルは `~/.config/opencode/skills/` にインストールされます。
     </TabItem>
   </Tabs>

3. インストールを確認：

   ```bash
   ls ~/.claude/skills/mutsumi-*/SKILL.md   # Claude Code の例
   # mutsumi-manage/SKILL.md
   # mutsumi-track/SKILL.md
   # mutsumi-plan/SKILL.md
   # mutsumi-report/SKILL.md
   # mutsumi-context/SKILL.md
   ```

4. 使い始める — Agent に話しかけるだけ：

   ```
   "ログインバグ修正のタスクを追加して、優先度 high、スコープ today"
   "認証リファクタリングをサブタスクに分解して"
   "タスクのステータスレポートを出して"
   ```

</Steps>

<Aside type="tip">
`mutsumi setup` は何度実行しても安全です。古いシンボリックリンクを置き換え、重複インストールは発生しません。
</Aside>

## 5 つのバンドルスキル

<CardGrid>
  <Card title="mutsumi-manage" icon="pencil">
    **タスク CRUD**：タスクの追加・編集・完了・削除。CLI コマンド（`mutsumi add`、`mutsumi done`）と直接 JSON 操作の両方に対応。
  </Card>
  <Card title="mutsumi-track" icon="approve-check">
    **自動進捗トラッキング**：作業中にタスクのステータスを自動更新。サブタスクの完了マークや親タスクの進捗更新を行う。Agent が自動的にトリガーし、手動呼び出し不要。
  </Card>
  <Card title="mutsumi-plan" icon="list-format">
    **タスク分解**：複雑な目標を parent-children 構造のアクショナブルなサブタスクに分解。呼び出し方："v2 への移行を計画して"。
  </Card>
  <Card title="mutsumi-report" icon="document">
    **ステータスレポート**：`mutsumi.json` を読み取り、スコープ別（Today → Week → Month → Inbox）の進捗サマリーを生成。呼び出し方："タスクレポートを出して"。
  </Card>
  <Card title="mutsumi-context" icon="open-book">
    **コンテキストローダー**：セッション開始時にタスクボードを読み込み、現在の優先順位を把握。Agent がサイレントに読み込む — 出力なし、認識のみ。自動トリガー、手動呼び出し不要。
  </Card>
</CardGrid>

## セットアップモード

| モード | コマンド | 動作 |
|--------|---------|------|
| **skills**（デフォルト） | `mutsumi setup --agent claude-code` | 5 つのスキルファイルをシンボリックリンクで Agent のスキルディレクトリにインストール。 |
| **skills+project-doc** | `mutsumi setup --agent claude-code --mode skills+project-doc` | スキルをインストールし、Agent のプロジェクトドキュメント（`CLAUDE.md`、`AGENTS.md` など）に統合手順を追記。 |
| **snippet** | `mutsumi setup --agent custom --mode snippet` | 統合プロンプトを標準出力に表示。ファイルのインストールなし。 |

### どのモードを使うべきか

- **skills** — ほとんどのユーザーに最適。Agent がスキルを自動検出します。
- **skills+project-doc** — リポジトリの Agent 設定ファイルにプロジェクトレベルの指示を追加。ルールをバージョン管理に含めたい場合に有用。
- **snippet** — Agent Skills 標準をサポートしないツール（Aider、カスタムスクリプトなど）向け。

## Agent がスキルを呼び出す方法

設定は不要です — Agent はスキルディレクトリをスキャンして自動検出します。

### ユーザー呼び出しスキル

関連するリクエストをすると発動：

| あなたの発言 | トリガーされるスキル |
|-------------|-------------------|
| 「ログインバグ修正のタスクを追加して」 | `mutsumi-manage` |
| 「リファクタリングをサブタスクに分解して」 | `mutsumi-plan` |
| 「全タスクのステータスレポートを出して」 | `mutsumi-report` |
| 「認証タスクを完了にして」 | `mutsumi-manage` |

### 自動トリガースキル

コンテキストに基づいて自動的に発動：

| 動作 | スキル |
|------|--------|
| Agent がセッション開始時にタスクボードを読み込む | `mutsumi-context` |
| Agent が作業中にタスク進捗を更新する | `mutsumi-track` |

<Aside>
自動トリガースキルは frontmatter に `user-invocable: false` が設定されています。バックグラウンドでサイレントに動作します。
</Aside>

## 対応 Agent 一覧

| Agent | スキルディレクトリ | プロジェクトドキュメント |
|-------|-------------------|----------------------|
| Claude Code | `~/.claude/skills/` | `CLAUDE.md` |
| Codex CLI | `~/.agents/skills/` | `AGENTS.md` |
| Gemini CLI | `~/.gemini/skills/` | `GEMINI.md` |
| OpenCode | `~/.config/opencode/skills/` | `opencode.md` |
| Aider | — | — |
| Custom | — | — |

<Aside type="note">
Aider とカスタム Agent は Agent Skills 標準をサポートしていません。`--mode snippet` を使用し、手動でプロンプトをペーストしてください。
</Aside>

## 確認とトラブルシューティング

```bash
# インストール済みスキルを確認
ls -la ~/.claude/skills/mutsumi-*/SKILL.md

# 再インストール（壊れたシンボリックリンクを修復）
mutsumi setup --agent claude-code

# アンインストール（手動削除）
rm -rf ~/.claude/skills/mutsumi-*
```
