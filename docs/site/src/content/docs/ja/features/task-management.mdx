## 概要

Mutsumi の TUI はフル CRUD に対応しています。外部 Agent なしでも、TUI 単体でタスク管理が完結します。

| アクション | マウス | キーボード |
|---|---|---|
| 作成 | フッターの `[+New]` ボタン | <kbd>n</kbd> → ポップアップ入力 |
| 閲覧 | タスク行をクリック | <kbd>Enter</kbd> で展開 |
| 編集 | タイトルをダブルクリック | <kbd>e</kbd> で編集 |
| 削除 | 右クリックメニュー → 削除 | <kbd>dd</kbd>（vim スタイル） |
| 完了 | チェックボックスをクリック | <kbd>Space</kbd> |
| 移動 | — | <kbd>J</kbd> / <kbd>K</kbd> で上下移動 |

## タスクの作成

<kbd>n</kbd> キーまたは `[+New]` ボタンで新しいタスクダイアログを開きます。

```
┌─────────── New Task ───────────┐
│ Title: [                     ] │
│ Scope: (Day) Week Month Inbox  │
│ Priority: High (Normal) Low    │
│ Tags: [                     ]  │
│                                │
│        [Create]  [Cancel]      │
└────────────────────────────────┘
```

- **タイトル**のみが必須フィールドです
- **Scope** はデフォルトで現在アクティブなタブに設定されます
- **Priority** はデフォルトで Normal です
- **Tags** はカンマ区切りで入力します
- **ID** と **created_at** は作成時に自動生成されます（UUIDv7）

## タスクの編集

2 つの編集モード：

### インライン編集（<kbd>i</kbd> キー）

タイトルがリスト行内で直接編集可能なテキストフィールドになります。

- <kbd>Enter</kbd> で確定
- <kbd>Escape</kbd> でキャンセル
- 変更は即座に JSON に書き戻されます

### フォーム編集（<kbd>e</kbd> キー）

すべてのフィールドを編集できるフォームダイアログが開きます。

## タスクの削除

<kbd>dd</kbd> キー（vim スタイル）またはコンテキストメニューでタスクを削除します。

- 確認ダイアログが表示されます（<kbd>y</kbd> で確認）
- 削除後、タスクは JSON から除去されます
- `task_deleted` イベントが発行されます

<Aside type="caution">
削除は元に戻せません。ただし、`tasks.json` を Git で管理していれば、`git checkout` で復元可能です。
</Aside>

## ステータスの切り替え

<kbd>Space</kbd> キーまたはチェックボックスのクリックでステータスを切り替えます。

- `pending` → `done`: `completed_at` が自動設定されます
- `done` → `pending`: `completed_at` がクリアされます
- 即座に JSON に書き戻され、確認ステップなし（サクサク感）

## 検索

<kbd>/</kbd> キーで検索モードに入ります。

```
┌─────────────────────────────────────────┐
│  🔍 Search: auth mod█                   │
├─────────────────────────────────────────┤
│  [ ] Auth モジュールをリファクタ  dev ★★★│  ← マッチした結果がハイライト
│  [x] Auth トークン期限切れを修正 bugfix  │
└─────────────────────────────────────────┘
```

- **リアルタイムフィルタリング**: 入力するたびにフィルタ
- **検索対象**: タイトル + タグ + description
- <kbd>Escape</kbd> で検索を終了し、フルリストに戻る

## ソート

<kbd>s</kbd> キーでソートモードに入ります。以下の基準でソート可能：

- 優先度（デフォルト）
- 作成日時
- 期日
- タイトル（アルファベット順）

## コピー & ペースト

| キー | アクション |
|---|---|
| <kbd>y</kbd> | タスクをコピー |
| <kbd>p</kbd> | カーソル位置の下にペースト |
| <kbd>P</kbd> (<kbd>Shift+P</kbd>) | カーソル位置の上にペースト |

## 優先度の変更

| キー | アクション |
|---|---|
| `+` / `=` | 優先度を上げる |
| `-` / `_` | 優先度を下げる |

## サブタスクの作成

<kbd>A</kbd>（<kbd>Shift+A</kbd>）でサブタスクを追加します。選択中のタスクの子タスクとして作成されます。

TUI では `[+Sub]` ボタンも詳細パネルに表示されます。

## 折りたたみ

- <kbd>z</kbd> キーでサブタスクの表示/非表示を切り替え
- <kbd>h</kbd> / <kbd>l</kbd> キー（vim プリセット）で優先度グループの折りたたみ/展開

<Aside type="note">
すべての CRUD 操作は即座に `tasks.json` に書き戻されます。TUI はステートレスで、常にファイルの状態を反映します。
</Aside>
