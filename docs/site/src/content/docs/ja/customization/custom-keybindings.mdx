## 概要

Mutsumi では、プリセット（vim/emacs/arrows）をベースに、個別のキーバインドをオーバーライドできます。

## key_overrides の設定

`config.toml` の `key_overrides` セクションで、アクション名に新しいキーを割り当てます：

```toml
keybindings = "vim"    # ベースプリセット

[key_overrides]
quit = "ctrl+q"        # 'q' の代わりに Ctrl+Q で終了
cursor_down = "ctrl+j" # 'j' の代わりに Ctrl+J で下に移動
cursor_up = "ctrl+k"   # 'k' の代わりに Ctrl+K で上に移動
```

## オーバーライド可能なアクション

以下のアクション名を使用して、任意のキーバインドをオーバーライドできます：

### ナビゲーション

| アクション名 | 説明 | vim デフォルト |
|---|---|---|
| `cursor_down` | カーソルを下に移動 | <kbd>j</kbd> |
| `cursor_up` | カーソルを上に移動 | <kbd>k</kbd> |
| `cursor_top` | リストの先頭にジャンプ | <kbd>gg</kbd> |
| `cursor_bottom` | リストの末尾にジャンプ | <kbd>G</kbd> |

### タスク操作

| アクション名 | 説明 | vim デフォルト |
|---|---|---|
| `toggle_done` | 完了/未完了の切り替え | <kbd>space</kbd> |
| `new_task` | 新しいタスクの作成 | <kbd>n</kbd> |
| `edit_task` | タスクの編集 | <kbd>e</kbd> |
| `inline_edit` | インライン編集 | <kbd>i</kbd> |
| `show_detail` | 詳細パネルの展開 | <kbd>enter</kbd> |
| `close_detail` | パネルを閉じる | <kbd>escape</kbd> |
| `priority_up` | 優先度を上げる | `+` |
| `priority_down` | 優先度を下げる | `-` |
| `copy_task` | タスクをコピー | <kbd>y</kbd> |
| `paste_task` | ペースト（下） | <kbd>p</kbd> |
| `paste_task_above` | ペースト（上） | <kbd>P</kbd> |
| `add_child` | サブタスクの追加 | <kbd>A</kbd> |
| `toggle_fold` | サブタスクの折りたたみ | <kbd>z</kbd> |
| `sort` | ソート | <kbd>s</kbd> |

### タブ操作

| アクション名 | 説明 | vim デフォルト |
|---|---|---|
| `next_tab` | 次のタブへ | <kbd>tab</kbd> |
| `prev_tab` | 前のタブへ | <kbd>shift+tab</kbd> |
| `tab_1` | Today タブ | `1` |
| `tab_2` | Week タブ | `2` |
| `tab_3` | Month タブ | `3` |
| `tab_4` | Inbox タブ | `4` |

### グループ操作

| アクション名 | 説明 | vim デフォルト |
|---|---|---|
| `collapse_group` | グループを折りたたみ | <kbd>h</kbd> |
| `expand_group` | グループを展開 | <kbd>l</kbd> |

### 移動操作

| アクション名 | 説明 | vim デフォルト |
|---|---|---|
| `move_down` | タスクを下に移動 | <kbd>J</kbd> |
| `move_up` | タスクを上に移動 | <kbd>K</kbd> |

### その他

| アクション名 | 説明 | vim デフォルト |
|---|---|---|
| `quit` | Mutsumi を終了 | <kbd>q</kbd> |
| `search` | 検索 | <kbd>/</kbd> |
| `show_help` | ヘルプ画面 | <kbd>?</kbd> |

## カスタムキーマップファイル

`~/.config/mutsumi/keys/custom.toml` にフルカスタムのキーマップを定義することもできます：

```toml
[keys]
move_down = "j"
move_up = "k"
toggle_done = "space"
new_task = "n"
edit_task = "e"
delete_task = "d,d"    # コード: 2 回キー押下
quit = "q"
search = "/"
help = "?"
tab_next = "tab"
tab_prev = "shift+tab"
tab_1 = "1"
tab_2 = "2"
tab_3 = "3"
tab_4 = "4"
expand = "enter"
top = "g"
bottom = "shift+g"
```

## キー記法

| 記法 | 説明 | 例 |
|---|---|---|
| 単一キー | そのままの文字 | <kbd>j</kbd>, <kbd>k</kbd>, <kbd>n</kbd> |
| 修飾キー | <kbd>ctrl+</kbd>, <kbd>shift+</kbd>, <kbd>alt+</kbd> | <kbd>ctrl+q</kbd>, <kbd>shift+g</kbd> |
| 特殊キー | キー名 | <kbd>space</kbd>, <kbd>enter</kbd>, <kbd>escape</kbd>, <kbd>tab</kbd> |
| コード | カンマ区切り | `d,d`（dd シーケンス） |

<Aside type="tip">
オーバーライドは**プリセットの上に重ねて適用**されます。オーバーライドしなかったキーはプリセットのデフォルトが維持されます。
</Aside>
