## Overview

Mutsumi offers three keybinding presets (vim, emacs, arrows). If you want to change individual keys without creating a full preset, use `key_overrides` in `config.toml`.

## Configuration

Add a `[key_overrides]` section to your `config.toml`:

```toml
keybindings = "vim"

[key_overrides]
quit = "ctrl+q"
cursor_down = "ctrl+j"
cursor_up = "ctrl+k"
```

Overrides are applied on top of the selected preset. Only the specified actions are reassigned; all other keys remain at their preset defaults.

## Action Names

These are all the action names you can override:

### Navigation

| Action | Description | vim default | emacs default | arrows default |
|--------|-------------|-------------|---------------|----------------|
| `cursor_down` | Move cursor down | <kbd>j</kbd> | <kbd>ctrl+n</kbd> | <kbd>down</kbd> |
| `cursor_up` | Move cursor up | <kbd>k</kbd> | <kbd>ctrl+p</kbd> | <kbd>up</kbd> |
| `cursor_top` | Jump to top | <kbd>gg</kbd> (multi-key) | <kbd>ctrl+a</kbd> | <kbd>home</kbd> |
| `cursor_bottom` | Jump to bottom | <kbd>G</kbd> | <kbd>ctrl+e</kbd> | <kbd>end</kbd> |

### Task Operations

| Action | Description | Default (all presets) |
|--------|-------------|----------------------|
| `toggle_done` | Toggle done/pending | <kbd>space</kbd> |
| `new_task` | Create new task | <kbd>n</kbd> |
| `edit_task` | Edit selected task | <kbd>e</kbd> |
| `inline_edit` | Inline edit title | <kbd>i</kbd> |
| `show_detail` | Open detail panel | <kbd>enter</kbd> |
| `close_detail` | Close detail panel | <kbd>escape</kbd> |
| `copy_task` | Copy task | <kbd>y</kbd> |
| `paste_task` | Paste below | <kbd>p</kbd> |
| `paste_task_above` | Paste above | <kbd>P</kbd> (shift+p) |
| `add_child` | Add subtask | <kbd>A</kbd> (shift+a) |
| `search` | Open search | <kbd>/</kbd> |
| `sort` | Open sort overlay | <kbd>s</kbd> |

### Priority

| Action | Description | Default |
|--------|-------------|---------|
| `priority_up` | Increase priority | `+` or `=` |
| `priority_down` | Decrease priority | `-` or `_` |

### View Controls

| Action | Description | vim default | emacs default | arrows default |
|--------|-------------|-------------|---------------|----------------|
| `collapse_group` | Collapse priority group | <kbd>h</kbd> | <kbd>ctrl+b</kbd> | <kbd>left</kbd> |
| `expand_group` | Expand priority group | <kbd>l</kbd> | <kbd>ctrl+f</kbd> | <kbd>right</kbd> |
| `toggle_fold` | Toggle subtask fold | <kbd>z</kbd> | <kbd>z</kbd> | <kbd>z</kbd> |
| `move_down` | Move task down | <kbd>J</kbd> | <kbd>ctrl+shift+n</kbd> | <kbd>shift+down</kbd> |
| `move_up` | Move task up | <kbd>K</kbd> | <kbd>ctrl+shift+p</kbd> | <kbd>shift+up</kbd> |

### Tab Management

| Action | Description | Default |
|--------|-------------|---------|
| `next_tab` | Next scope tab | <kbd>tab</kbd> |
| `prev_tab` | Previous scope tab | <kbd>shift+tab</kbd> |
| `tab_1` | Switch to Today | `1` |
| `tab_2` | Switch to Week | `2` |
| `tab_3` | Switch to Month | `3` |
| `tab_4` | Switch to Inbox | `4` |

### Utility

| Action | Description | vim/arrows default | emacs default |
|--------|-------------|-------------------|---------------|
| `quit` | Quit Mutsumi | <kbd>q</kbd> | <kbd>ctrl+q</kbd> |
| `show_help` | Show help screen | <kbd>?</kbd> | <kbd>?</kbd> |

## Key Syntax

Key strings follow Textual's key notation:

| Example | Meaning |
|---------|---------|
| <kbd>j</kbd> | Single character key |
| <kbd>space</kbd> | Space bar |
| <kbd>enter</kbd> | Enter key |
| <kbd>escape</kbd> | Escape key |
| <kbd>tab</kbd> | Tab key |
| <kbd>ctrl+q</kbd> | Ctrl + Q |
| <kbd>shift+tab</kbd> | Shift + Tab |
| <kbd>ctrl+shift+n</kbd> | Ctrl + Shift + N |
| <kbd>up</kbd> | Up arrow |
| <kbd>down</kbd> | Down arrow |
| <kbd>left</kbd> | Left arrow |
| <kbd>right</kbd> | Right arrow |
| <kbd>home</kbd> | Home key |
| <kbd>end</kbd> | End key |
| `slash` | <kbd>/</kbd> key |
| `question_mark` | <kbd>?</kbd> key |

## Example: Hybrid vim + Ctrl-Q quit

```toml
keybindings = "vim"

[key_overrides]
quit = "ctrl+q"
```

## Example: Arrow keys with vim-style J/K reorder

```toml
keybindings = "arrows"

[key_overrides]
move_down = "J"
move_up = "K"
```

<Aside type="caution">
Multi-key sequences (<kbd>dd</kbd>, <kbd>gg</kbd>) are handled by the KeyManager engine, not the Textual Binding system. Overriding `cursor_top` changes the single-key binding but does not affect the <kbd>gg</kbd> multi-key sequence in the vim preset.
</Aside>

## How Overrides Work

Under the hood, `get_keybindings()` takes the preset bindings and replaces the key for any action listed in `key_overrides`:

```python
# config.toml
[key_overrides]
quit = "ctrl+q"

# Result: the "q" binding for quit is replaced with "ctrl+q"
# All other bindings remain unchanged
```

The override mechanism operates at the Textual `Binding` level. The action name, description, and show behavior are preserved; only the key string is swapped.
