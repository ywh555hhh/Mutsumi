## 爽点 1：开箱即用

**目标**：从零到看到自己的第一个任务 — 不用翻文档，不用写配置，不用 YAML。

### 前置条件

| 要求 | 检查方式 |
|---|---|
| macOS / Linux 终端 | `uname` |
| Python 3.12+ | `python3 --version` |
| `uv` 或 `pip` | `uv --version` 或 `pip --version` |

<Aside type="note">
Windows 用户：请使用 WSL 或 PowerShell。原生 Windows 可以用，但不是本次内测的主要目标。
</Aside>

### 第一步：安装

<Aside type="caution">
  v1.0.0b1 是**预发布版本**，包管理器默认跳过预发布 — 必须显式指定。
  需要 **Python 3.12+**，用 `python3 --version` 检查。
</Aside>

<Tabs>
  <TabItem label="uv（推荐）">
  ```bash
  uv tool install --prerelease allow mutsumi-tui
  ```
  </TabItem>
  <TabItem label="pip">
  ```bash
  pip install --pre mutsumi-tui
  # 或锁定精确版本：
  pip install mutsumi-tui==1.0.0b1
  ```
  </TabItem>
  <TabItem label="pipx">
  ```bash
  pipx install mutsumi-tui --pip-args='--pre'
  ```
  </TabItem>
  <TabItem label="从源码">
  ```bash
  uv tool install git+https://github.com/ywh555hhh/Mutsumi.git
  ```
  </TabItem>
</Tabs>

验证：

```bash
mutsumi --version   # → mutsumi, version 1.0.0b1
mutsumi --help      # → 显示子命令列表
```

### 第二步：首次启动

```bash
cd ~/some-project
mutsumi
```

你应该看到一个**单页 onboarding 界面**：

```
┌─────────────────────────────────────────────────────────┐
│               Welcome to Mutsumi                        │
│                                                         │
│  Language       ● English  ○ 中文  ○ 日本語             │

│  Theme          ● Monochrome Zen  ○ Nord  ○ Dracula     │
│  Workspace      ● Personal + Project                    │
│  Agent          ○ Claude Code  ○ Codex CLI  ● Skip      │
│                                                         │
│           [Start Mutsumi]           [Skip]               │
└─────────────────────────────────────────────────────────┘
```

<Aside type="tip">
试试选择 **中文 + Nord + Vim**，验证语言、主题、快捷键是否全部即时生效。
</Aside>

### 第三步：创建你的第一个任务

onboarding 完成后你会看到一个空看板。点击 **[+ New Task]** 或按 <kbd>n</kbd>，输入标题，点 Create。

### 检查清单

| # | 检查项 | 预期结果 |
|---|---|---|
| 1 | 安装命令正常 | 纯 Python，无需编译 |
| 2 | `mutsumi --version` | `1.0.0b1` |
| 3 | 首次启动显示 onboarding | 单页表单，不是多步向导 |
| 4 | 5 个设置项全部可见 | 语言、快捷键、主题、工作区、Agent |
| 5 | 选中文 → Start | UI 立即变成中文 |
| 6 | 选 Nord → Start | 颜色立即变化 |
| 7 | 点 Skip | 默认设置进入 TUI，不崩溃 |
| 8 | 第二次启动跳过 onboarding | 直接进 TUI |
| 9 | 空状态有 [+ New Task] | 可点击的按钮 |
| 10 | 按 `?` | 显示帮助页面 |

**通过标准**：从零到第一个任务，60 秒以内，不需要翻任何文档。

## 爽点 2：Agent 实时同步

**目标**：你随口跟 Agent 说话，Agent 写 JSON，Mutsumi 在 Agent 还没打完回复时就已经更新了。

### 准备：分屏终端

你需要左右两个窗格：

```
┌──────────────────────────┬──────────────────────────┐
│                          │                          │
│   AI Agent               │   Mutsumi TUI            │
│   (Claude Code 等)       │                          │
│                          │                          │
└──────────────────────────┴──────────────────────────┘
```

<Tabs>
  <TabItem label="tmux">
  ```bash
  tmux new-session -d -s dev
  tmux split-window -h -p 35 "mutsumi"
  tmux select-pane -t 0
  tmux attach -t dev
  ```
  </TabItem>
  <TabItem label="iTerm2">
  按 <kbd>Cmd+D</kbd> 分屏。右边：`mutsumi`。左边：你的 agent。
  </TabItem>
  <TabItem label="VS Code / Cursor">
  拆分终端面板。右边：`mutsumi`。左边：你的 agent。
  </TabItem>
</Tabs>

### Skill 注入

让 Agent 认识 Mutsumi 的 JSON 协议：

```bash
mutsumi setup --agent claude-code    # 或 codex-cli, gemini-cli, opencode
```

<Aside>
如果你在 onboarding 时已经选了 Agent，这步会自动完成。
</Aside>

### 核心循环：说 → 写 → 看

试试这些对话：

<Steps>

1. **随口加个任务**

   ```
   你："帮我加个 todo，明天交周报"
   ```

   Agent 写入 `mutsumi.json`。Mutsumi 检测到变化，新任务出现 — 无需重启，无闪烁。

2. **批量添加**

   ```
   你："把这三个 bug 加进去：登录页白屏、支付超时、头像上传失败，都是 high priority"
   ```

   3 个任务一次性出现。

3. **标记完成**

   ```
   你："登录页白屏修好了，标记完成"
   ```

   复选框更新，任务变灰。

4. **修改字段**

   ```
   你："把支付超时的优先级降到 normal，加个 tag 叫 backend"
   ```

   优先级星星和标签立即更新。

</Steps>

### 反向同步

同步是**双向的**。在 Mutsumi 里勾选完成 → `mutsumi.json` 更新 → Agent 可以读取最新状态。

### 检查清单

| # | 检查项 | 预期结果 |
|---|---|---|
| 1 | Agent 添加 1 个任务 | 1 秒内出现 |
| 2 | Agent 一次加 3 个任务 | 全部出现，一次重渲染 |
| 3 | Agent 标记完成 | 复选框 + 删除线更新 |
| 4 | Agent 改优先级 | 星星立即变化 |
| 5 | Agent 写入无效 JSON | 显示错误横幅，不崩溃 |
| 6 | Agent 修复 JSON | 错误消失，任务恢复 |
| 7 | 更新过程无闪烁 | 光标位置保持不变 |
| 8 | 在 TUI 中勾选完成 | JSON 文件更新，Agent 可以读到 |

**通过标准**：任务在 Agent 还没打完回复时就出现了。你的反应：「哦这也太爽了吧」。
