## 工作原理

Mutsumi 内置 **5 个技能文件**，遵循 [Agent Skills 开放标准](https://agentskills.io)。运行 `mutsumi setup` 时，这些技能文件通过符号链接安装到你的 Agent 技能目录中。Agent 会自动发现它们 —— 无需手动粘贴 prompt。

```
~/.mutsumi/skills/          ← SSOT（唯一真实来源）
  mutsumi-manage/SKILL.md
  mutsumi-track/SKILL.md
  mutsumi-plan/SKILL.md
  mutsumi-report/SKILL.md
  mutsumi-context/SKILL.md
        │
        ├── symlink → ~/.claude/skills/     (Claude Code)
        ├── symlink → ~/.agents/skills/     (Codex CLI)
        ├── symlink → ~/.gemini/skills/     (Gemini CLI)
        └── symlink → ~/.config/opencode/skills/  (OpenCode)
```

## 快速开始

<Steps>

1. 安装 Mutsumi（如尚未安装）：

   ```bash
   pip install mutsumi-tui
   ```

2. 为你的 Agent 运行 setup：

   <Tabs>
     <TabItem label="Claude Code">
     ```bash
     mutsumi setup --agent claude-code
     ```
     技能安装到 `~/.claude/skills/`。
     </TabItem>
     <TabItem label="Codex CLI">
     ```bash
     mutsumi setup --agent codex-cli
     ```
     技能安装到 `~/.agents/skills/`。
     </TabItem>
     <TabItem label="Gemini CLI">
     ```bash
     mutsumi setup --agent gemini-cli
     ```
     技能安装到 `~/.gemini/skills/`。
     </TabItem>
     <TabItem label="OpenCode">
     ```bash
     mutsumi setup --agent opencode
     ```
     技能安装到 `~/.config/opencode/skills/`。
     </TabItem>
   </Tabs>

3. 验证安装：

   ```bash
   ls ~/.claude/skills/mutsumi-*/SKILL.md   # 以 Claude Code 为例
   # mutsumi-manage/SKILL.md
   # mutsumi-track/SKILL.md
   # mutsumi-plan/SKILL.md
   # mutsumi-report/SKILL.md
   # mutsumi-context/SKILL.md
   ```

4. 开始使用 —— 告诉你的 Agent：

   ```
   "添加一个任务：修复登录 Bug，优先级 high，范围 today"
   "把 auth 重构拆分成子任务"
   "给我一份任务状态报告"
   ```

</Steps>

<Aside type="tip">
重复运行 `mutsumi setup` 是安全的 —— 会替换过期的符号链接，不会重复安装。
</Aside>

## 5 个内置技能

<CardGrid>
  <Card title="mutsumi-manage" icon="pencil">
    **任务增删改查**：添加、编辑、完成、删除任务。支持 CLI 命令（`mutsumi add`、`mutsumi done`）和直接 JSON 操作。
  </Card>
  <Card title="mutsumi-track" icon="approve-check">
    **自动进度跟踪**：在你工作时自动更新任务状态。标记子任务完成、更新父任务进度。由 Agent 自动触发，非手动调用。
  </Card>
  <Card title="mutsumi-plan" icon="list-format">
    **任务分解**：将复杂目标分解为可执行的子任务，使用 parent-children 结构。调用方式："帮我规划 v2 迁移方案"。
  </Card>
  <Card title="mutsumi-report" icon="document">
    **状态报告**：读取 `mutsumi.json`，生成按范围分组的进度摘要（Today → Week → Month → Inbox）。调用方式："给我一份任务报告"。
  </Card>
  <Card title="mutsumi-context" icon="open-book">
    **上下文加载器**：在会话开始时加载任务板，理解当前优先级。Agent 静默读取 —— 没有输出，只有感知。自动触发，非手动调用。
  </Card>
</CardGrid>

## Setup 模式

| 模式 | 命令 | 行为 |
|------|------|------|
| **skills**（默认） | `mutsumi setup --agent claude-code` | 将 5 个技能文件以符号链接安装到 Agent 的技能目录。 |
| **skills+project-doc** | `mutsumi setup --agent claude-code --mode skills+project-doc` | 安装技能，并在 Agent 项目文档（`CLAUDE.md`、`AGENTS.md` 等）中追加集成说明。 |
| **snippet** | `mutsumi setup --agent custom --mode snippet` | 将集成 prompt 打印到标准输出，供手动粘贴。不安装任何文件。 |

### 何时使用哪种模式

- **skills** —— 适合大多数用户。Agent 会自动发现技能文件。
- **skills+project-doc** —— 在仓库的 Agent 配置文件中添加项目级说明。适合希望把规则纳入版本控制的场景。
- **snippet** —— 用于不支持 Agent Skills 标准的工具（如 Aider、自定义脚本）。

## Agent 如何调用技能

你不需要做任何配置 —— Agent 通过扫描其技能目录自动发现技能。

### 用户可调用的技能

当你提出相关请求时触发：

| 你说的话 | 触发的技能 |
|----------|-----------|
| "添加一个任务：修复登录 Bug" | `mutsumi-manage` |
| "把重构任务拆分成子任务" | `mutsumi-plan` |
| "给我一份所有任务的状态报告" | `mutsumi-report` |
| "把 auth 任务标记为完成" | `mutsumi-manage` |

### 自动触发的技能

这些技能根据上下文自动激活：

| 行为 | 技能 |
|------|------|
| Agent 在会话开始时读取任务板 | `mutsumi-context` |
| Agent 在工作过程中更新任务进度 | `mutsumi-track` |

<Aside>
自动触发的技能在 frontmatter 中设有 `user-invocable: false`。它们在后台静默工作。
</Aside>

## 支持的 Agent

| Agent | 技能目录 | 项目文档 |
|-------|---------|---------|
| Claude Code | `~/.claude/skills/` | `CLAUDE.md` |
| Codex CLI | `~/.agents/skills/` | `AGENTS.md` |
| Gemini CLI | `~/.gemini/skills/` | `GEMINI.md` |
| OpenCode | `~/.config/opencode/skills/` | `opencode.md` |
| Aider | — | — |
| Custom | — | — |

<Aside type="note">
Aider 和自定义 Agent 不支持 Agent Skills 标准。请使用 `--mode snippet` 并手动粘贴 prompt。
</Aside>

## 验证与排障

```bash
# 检查已安装的技能
ls -la ~/.claude/skills/mutsumi-*/SKILL.md

# 重新安装（修复损坏的符号链接）
mutsumi setup --agent claude-code

# 卸载（手动删除）
rm -rf ~/.claude/skills/mutsumi-*
```
